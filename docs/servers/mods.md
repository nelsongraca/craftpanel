# Mods & Plugins

Mod management is available for server types that support it: Fabric, Forge, NeoForge, Quilt, Paper, Spigot. Plugin management for proxy types follows the same flow where supported.

Requires `server.mods` permission.

## Adding Mods

1. Open the **Mods** tab on a server's detail page
2. Search [Modrinth](https://modrinth.com) directly from the panel — results are automatically filtered by the server's loader and Minecraft version
3. Select a mod to add it to the server's mod list
4. Choose a version pin strategy (see below)
5. The change is saved immediately; it takes effect on the next restart

## Version Pinning

Each mod entry has two options:

| Strategy | Behaviour |
|---|---|
| **Pinned** | A specific version is locked. itzg downloads exactly that version on restart. Nothing changes until the pin is explicitly updated. |
| **Latest** | itzg downloads the latest compatible version on every restart. Updates happen automatically but may introduce breaking changes. |

!!! tip
    Pinned is the recommended default for production servers. Use Latest only if you want automatic updates and are prepared to handle occasional compatibility issues.

## Updating Mods

The mod list UI shows the installed version and the latest available version for each mod. A one-click **Update** action updates the pin to the latest version and marks the server for a restart.

## How It Works

The mod list is persisted in the master database and passed to the itzg container via the `MODRINTH_PROJECTS` environment variable. itzg handles the actual download and placement into the `mods/` directory on container startup.

Mod config files (generated by mods on first run) are accessible via the [File Explorer](./files-console.md).
